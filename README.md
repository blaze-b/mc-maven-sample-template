Sorting Algorithms

Comparison-Based Sorting:

Bubble Sort: Simple but inefficient.
Insertion Sort: Efficient for small datasets or nearly sorted data.
Selection Sort: Simple but inefficient.
Merge Sort: Divide and conquer, stable sort, O(n log n) time complexity.
Quick Sort: Divide and conquer, in-place sort, average O(n log n) time complexity, worst-case O(n^2).
Heap Sort: Uses a heap data structure, O(n log n) time complexity.
Non-Comparison-Based Sorting:

Counting Sort: Efficient for small ranges of integers.
Radix Sort: Sorts based on digits or characters.
Bucket Sort: Distributes elements into buckets and then sorts the buckets.
Sorting Algorithm Analysis: Time complexity (best, average, worst case), space complexity, stability.

4. Searching Algorithms

Linear Search: Sequential search, simple but inefficient for large datasets.
Binary Search: Efficient for sorted data, logarithmic time complexity.
Depth-First Search (DFS): For searching trees and graphs.
Breadth-First Search (BFS): For searching trees and graphs.
Other Search Techniques:
Hashing (for near constant-time lookups).
Trie search (for prefix-based searching).
Search Algorithm Analysis: Time complexity, space complexity.
